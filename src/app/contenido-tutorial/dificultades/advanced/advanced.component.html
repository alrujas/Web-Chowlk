Class Descriptions

A class description in OWL serves to define or describe a class within an ontology. These descriptions can take various forms, each serving a distinct purpose in specifying the characteristics and constraints of the class. OWL identifies six primary types of class descriptions:

  • Class Identifier: This type of description specifies a class by its unique URI reference, essentially associating a class name with its identity within the ontology.
  • Exhaustive Enumeration: Here, the class is defined by enumerating all the individual instances that belong to it. This exhaustive list collectively represents the instances of the class.
  • Property Restriction: In this type of description, constraints are imposed on the properties of instances belonging to the class. These restrictions help refine the definition of the class by specifying certain property-value combinations that its instances must satisfy.
  • Intersection: An intersection description involves combining two or more class descriptions to define a new class that includes only those instances that satisfy all the constraints imposed by each of the intersected classes.
  • Union: Conversely, a union description combines multiple class descriptions to define a new class that includes instances that satisfy the constraints of any of the unioned classes.
  • Complement: This type of description defines a class by specifying the instances that do not belong to another class. It essentially represents the complement of the specified class.

While the first type uniquely identifies a class by its name through a URI reference, the remaining five types describe anonymous classes by imposing constraints on the class extension, thereby refining the definition of the class without explicitly naming it.

Enumeration

In OWL, enumeration refers to a method of defining a class by explicitly listing all its individual instances. This technique allows for precise specification of the members of a class, making it particularly useful for defining classes with a limited and well-defined set of instances.

Restriction

In OWL, a restriction on a property serves as a way to define a specific class of individuals based on certain conditions. There are two main types of property restrictions.

  • Value Constraint: This type of restriction sets boundaries or limitations on the possible values that a property can have when associated with instances of a particular class. Essentially, it defines what kind of values are allowed for that property within the context of the class being described.
  • Cardinality Constraint: A cardinality constraint, on the other hand, regulates the quantity of values that a property can have in relation to instances of a given class. It specifies how many instances or values of the property are required or allowed for individuals belonging to that class.

These property restrictions are versatile and can be applied to both datatype properties (properties with literal values such as strings or numbers) and object properties (properties linking individuals to other individuals). By using property restrictions, ontology designers can precisely define the characteristics and relationships of classes within the ontology, facilitating more accurate modeling of domain concepts.

Value Constraints

Value constraints in OWL allow for the specification of restrictions on the values that a property can have when associated with instances of a particular class. There are several types of value constraints provided by OWL, each serving a specific purpose.

owl:allValuesFrom Constraint:

The owl:allValuesFrom constraint links a restriction class to either a class description or a data range. It describes a class of all individuals for which all values of the property under consideration must be either members of the specified class or fall within the specified data range.

Cardinality Constraints

A cardinality constraint restricts the number of values that a property can take for individuals belonging to a specific class. OWL provides three types of cardinality constraints: owl:maxCardinality, owl:minCardinality, and owl:cardinality.

owl:maxCardinality :

The owl:maxCardinality constraint is a predefined OWL property that associates a restriction class with a numerical value from the nonNegativeInteger datatype in the XML Schema. When applied, this constraint defines a class comprising individuals that possess, at most, N unique values for the specified property. Here, N represents the value set by the cardinality constraint.

owl:minCardinality :

The owl:minCardinality constraint is a predefined OWL property that associates a restriction class with a numerical value from the nonNegativeInteger datatype in the XML Schema. When applied, this constraint defines a class comprising individuals that possess, at least, N unique values for the specified property. Here, N represents the value set by the cardinality constraint.

owl:cardinality :

The owl:cardinality constraint is a standard OWL property that connects a restriction class to a numerical value within the range of the XML Schema datatype nonNegativeInteger. When utilized, this constraint specifies a class consisting of individuals that possess precisely N unique values for the designated property. Here, N denotes the value established by the cardinality constraint.

Intersection

Intersection is an operation in OWL that allows us to define a new class by taking the common elements from two or more existing classes. It represents the set of individuals that are instances of all the classes involved in the intersection.

Suppose we have two classes: "Adults" and "Male." We can define a new class called "Adult Males" by taking the intersection of these two classes, representing individuals who are both adults and male.

Union

Union is another operation in OWL that allows us to define a new class by taking the combined elements from two or more existing classes. It represents the set of individuals that are instances of at least one of the classes involved in the union.

Consider two classes: "Birds" and "Mammals." We can define a new class called "Animals" by taking the union of these two classes, representing all individuals that are either birds or mammals

Complement

Complement is an operation in OWL that allows us to define a new class by excluding the elements of an existing class from the universe of discourse. It represents the set of individuals that are not instances of a particular class.

Suppose we have a class called "Non-Smokers" representing individuals who do not smoke. We can define a new class called "Smokers" as the complement of "Non-Smokers," representing individuals who smoke.

Enumerated Datatypes

OWL introduces an alternative way to specify a range of data values called an enumerated datatype. This type of datatype employs the owl:oneOf construct. When defining an enumerated datatype, a blank node of class owl:DataRange serves as the subject of owl:oneOf, while the object consists of a list of literals.

Moreover, costum data values can be defined specifying the prefixes of the datatypes. By default the prefix is xsd.
